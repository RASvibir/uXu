What is CyberCat Sunflower?

CyberCat Sunflower is a decentralized, community-driven archive platform built for music lovers who want to preserve and share live performance history. Instead of a centralized database controlled by one entity, we believe in distributed archives—where you can create, maintain, and share archives of your favorite artists.

Every archive is verified through GitHub, maintained by its creator, and accessible to the community. This repository serves as both a working archive (the Grateful Dead shows we're starting with) and a template for others to fork and create their own.

How to Contribute Your Archive

Follow these steps to create and submit your own music archive:

1. Fork the uXu Repository

Visit https://github.com/RASvibir/uXu and click "Fork" to create your own copy.

2. Create Your Archive Directory

In your fork, create: /archives/{YourArchiveName}/

3. Add Your Data File

Create a data.json file following this schema:

{ "archiveName": "Your Archive Name", "artist": "Artist Name", "curator": "Your Name", "description": "Description of this archive", "shows": [ { "date": "YYYY-MM-DD", "venue": "Venue Name", "city": "City", "state": "State", "country": "Country", "setlist": ["Song 1", "Song 2", "Song 3"], "notes": "Any notes about the show", "archiveOrgId": "identifier (optional, for audio)" } ] }

4. Create a README

In your archive folder, create a README.md describing your archive and how to use it.

5. Submit a Pull Request

Open a PR to the main uXu repo. Describe your archive and any audio sources you've linked. The maintainer will review and merge if it meets the guidelines.

6. Your Archive Goes Live

Once merged, your archive will appear on the main CyberCat Sunflower index and be accessible to the community!

Archive Requirements

  • ✓ Data must be publicly available (public domain or licensed for sharing)
  • ✓ Archive must include at least 10 shows/events
  • ✓ Your name/handle as curator for attribution
  • ✓ A description explaining the archive's purpose
  • ✓ Audio sources must be legal streaming (Internet Archive, Spotify API, etc.)

About This Project

Vision

CyberCat Sunflower exists because music history belongs to the community, not corporations. We believe in decentralized, verifiable archives that celebrate live performance culture. By using GitHub as our backbone, we maintain transparency and attribution—every archive creator is credited, every change is tracked, and the community owns the history.

Technology

Built with:

  • 📦 GitHub for version control & community submission
  • 🎵 Internet Archive API for audio streaming (highest quality available)
  • 🎨 Cyberpunk/Trippy Aesthetics for the vibe
  • ⚡ Vanilla JS + HTML/CSS for speed & accessibility

The Grateful Dead Archive

We're starting with the Grateful Dead—one of the most documented live bands in history. Their shows are public domain and well-documented, making them perfect for demonstrating how this platform works. But CyberCat Sunflower is designed to host archives for any artist.

Contribute

Want to add shows to an existing archive? Found an error? Open an issue or submit a PR on GitHub. This is community-driven archival.
