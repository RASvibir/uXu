# 📖 RTFM: Read The Friendly Manual

A searchable archive of **working code, templates, and instructions** from uXu contributors.

This is **not** a standard or requirement. It's a **resource**—if you want to build an interactive archive, learn from how others solved problems, or reference working examples, it's here.

## What's in RTFM?

### Code Examples
- **Audio players** — Interactive HTML/CSS/JS for streaming audio with controls
- **Gallery templates** — Image browser, slideshow, lightbox patterns
- **Data structures** — JSON templates for common archive types
- **Styling frameworks** — CSS for retro, modern, minimalist, experimental designs

### Guides & Documentation
- **Sourcing audio legally** — Internet Archive, Creative Commons, artist permissions
- **Building your first archive** — Step-by-step for different archive types
- **Metadata best practices** — Structuring setlists, venue info, historical context
- **Accessibility tips** — Making archives usable for everyone
- **Performance optimization** — Keeping archives fast even with lots of media

### Troubleshooting
- Common browser issues with Web Audio API
- Streaming audio from different sources
- Building responsive designs
- CORS and cross-origin media issues

## How to Search RTFM

Each file is tagged with keywords:

```
# Audio Player with Equalizer
Tags: audio, player, controls, web-audio-api, interactive
Source: CyberCat Sunflower archive
```

Browse by topic or keyword to find what you need.

## How to Contribute to RTFM

Found something that works? Share it:

1. Add a file with clear comments and documentation
2. Include tags at the top
3. Link to the original archive it came from (give credit)
4. Write a brief explanation of what it does

Example:
```
# Simple Audio Gallery
Tags: gallery, audio, responsive, beginners

From: [Your Archive Name]

## What this does
Displays a grid of audio files with titles and descriptions.
Click to play. Uses HTML5 audio element—no dependencies.

## How to use
1. Create an array of audio objects
2. Loop through and generate cards
3. Attach play listeners

## Code
[your code here]
```

## Respect & Attribution

- **Always credit** the archive or contributor where code came from
- **Explain modifications** if you adapt existing code
- **Keep licenses clear** — link to the original license
- **Don't remove attribution** when sharing examples

## Important Note

RTFM is **optional inspiration, not a rule book**. You don't have to use any of this. You can:
- Build from scratch
- Mix and match from different examples
- Ignore RTFM entirely and do your own thing

The archive exists to help, not to constrain.
